Linux Commands to Find Incoming and Outgoing Listening Sessions

netstat

Displays active connections, listening ports, and network statistics.

Example to find listening sessions:

netstat -tuln
  • -t: TCP connections.
  • -u: UDP connections.
  • -l: Listening ports.
  • -n: Show numerical addresses instead of resolving names.

ss (socket statistics)

A faster, modern alternative to netstat.

Example to find listening sessions:

ss -tuln

Similar options as netstat.

lsof (list open files)

Lists open files and network sockets.

Example to list network sessions:

lsof -i

Add -iTCP or -iUDP to filter specific protocols.

sudo netstat -plant

Provides detailed output, including PID and program name.

Example:

sudo netstat -plant | grep LISTEN

iptables or firewalld

Check firewall rules for open ports.

Example:

sudo iptables -L -n -v

Using grep

Definition:

grep is used to search for specific patterns in text files or output.

Basic Syntax:

grep [options] 'pattern' [file]

Examples:

Search for a specific word in a file:

grep "keyword" file.txt

Case-insensitive search:

grep -i "keyword" file.txt

Search recursively in directories:

grep -r "keyword" /path/to/directory

Match whole words only:

grep -w "keyword" file.txt

Show line numbers:

grep -n "keyword" file.txt

Using grep with Other Commands:

Filter netstat output:

netstat -tuln | grep LISTEN

Search processes with ps:

ps aux | grep process_name

Regular Expressions with grep:

Use grep -E for extended regular expressions.

Example: Find lines starting with "Error":

grep "^Error" file.txt
